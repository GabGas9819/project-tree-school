I link in HTML, l'ipertesto

  • L'ipertestualità è un concetto che si riferisce alla capacità di collegare diversi contenuti attraverso i link all'interno di una pagina web.
  • I link sono elementi essenziali delle pagine web e consentono agli utenti di spostarsi tra diverse risorse o pagine.
  • I link in una pagina web possono essere utilizzati per collegare diverse informazioni tra loro, creando una rete di connessioni ipertestuali.

Il web è costituito da ipertesti e non solo da testi. I link sono il mezzo per passare da un testo all'altro. I link sono formati da due componenti.

Il contenutoLa risorsa
È la parte visibile del link, e proprio per questo l'utente deve essere sempre in grado di capire quali sono i collegamenti da cliccare all'interno della pagina
  • Si tratta di un collegamento ipertestuale che può essere un link a un'altra pagina sullo stesso server o su un server diverso, oppure può essere un collegamento interno a un punto specifico della stessa pagina.
  • Questo tipo di collegamento consente agli utenti di navigare facilmente tra le pagine web o di spostarsi all'interno di una stessa pagina.
  • I collegamenti interni possono essere utilizzati per creare una migliore esperienza utente e facilitare la navigazione all'interno di un sito web.
"nasconde" il collegamento (non importa se si tratta di testo o di immagine)
  • Un collegamento è un elemento in HTML che permette di creare un collegamento ipertestuale tra una pagina e un'altra.
  • Il collegamento indica la direzione in cui punta, ossia verso quale pagina o risorsa si sta navigando.
  • Il collegamento può essere rappresentato da testo cliccabile o da un'immagine.

Si utilizza la metafora dell'ancora per spiegare i link, con "la testa" del link all'interno del documento stesso e la "coda" in un altro documento o punto del documento stesso. I link sono importanti per la navigazione tra pagine web e per creare una struttura ipertestuale.

Link che puntano ad altri documenti 

                    I contenuti per sviluppatori sono su
                    < a href="https://www.w3.org">w3c.org< /a >.

Che produce come risultato:

I contenuti per sviluppatori sono su w3c.org.

Come si può intuire la testa della nostra àncora è il testo w3c.org, mentre la coda, cioè la destinazione (specificata dall'attributo href) è il sito web verso cui il link punta, cioè https://www.w3.org.

Il meccanismo del link funziona allo stesso modo indipendentemente dal tipo di risorsa a cui punta. Il comportamento del browser può variare a seconda del tipo di risorsa. Ad esempio, è indifferente se la destinazione dell'ancora è una pagina HTML, un'immagine, un file pdf, un file zip o un file exe.

TipologiaComportamento standard
Immagine .gif, .jpg, .png Viene visualizzata nel browser
Documento .html, .pdf, .doc
  • La pagina web viene visualizzata nel browser.
  • Alcuni documenti come .doc e .pdf potrebbero richiedere l'installazione di un plugin apposito.
  • Se il plugin non è installato, il sistema chiederà all'utente se salvare il file.
File .zip, file .exe Viene chiesto all'utente di scaricare il file
Nota: per motivi di sicurezza il download di un file (specie se .exe) può essere bloccato da un antivirus o da un firewall.
